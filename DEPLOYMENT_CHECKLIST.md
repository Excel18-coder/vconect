## Environment Variables (Backend) ‚òëÔ∏è

Go to v-market-backend service ‚Üí Environment tab

### Required Variables (Add These):
- [ ] `DATABASE_URL` = Your Neon database URL
- [ ] `CLOUDINARY_CLOUD_NAME` = Your Cloudinary cloud name
- [ ] `CLOUDINARY_API_KEY` = Your Cloudinary API key
- [ ] `CLOUDINARY_API_SECRET` = Your Cloudinary API secret

### Auto-Generated (Verify These Exist):
- [ ] `JWT_SECRET` (auto-generated by Render)
- [ ] `REFRESH_TOKEN_SECRET` (auto-generated by Render)
- [ ] `NODE_ENV` = `production`
- [ ] `PORT` = `10000`
- [ ] `CLIENT_URL` = `https://v-market.onrender.com`

- [ ] Clicked "Save Changes"
- [ ] Waited for automatic redeploy (2-3 minutes)

## Troubleshooting (If Issues Occur) ‚òëÔ∏è

### If Build Fails:
- [ ] Check build logs in Render dashboard
- [ ] Verify all dependencies in package.json
- [ ] Check Node version compatibility
- [ ] Test build locally: `npm run build`

### If Service Won't Start:
- [ ] Check runtime logs for errors
- [ ] Verify all required env vars are set
- [ ] Check DATABASE_URL is correct
- [ ] Verify port configuration

### If CORS Errors:
- [ ] Check CLIENT_URL matches frontend URL
- [ ] Verify backend CORS config includes frontend
- [ ] Check both services use HTTPS

### If Database Errors:
- [ ] Verify Neon database is running
- [ ] Check connection string format
- [ ] Ensure pooling is enabled
- [ ] Test connection locally first
